1. 模型设置和算法步骤

模型设置与前面计算稳态的模型几乎一致，只是为了加快计算速度，把60期换成了6期。然后，我们计算当养老金替代率在\(t=0\)时为0.3，在\(t=1\)时下降到0.2，然后计算总量变量\(K_t,N_t\)如何从旧稳态抵达新稳态，这个变化的路径称为转移路径。

算法如下：

  1. 选择转轨的期数\(t_c\)，一般为\(3\times (T+T^R)\)会充分保证收敛到新稳态。\(T\)是工作时期，\(T^R\)是退休时期。
  2. 计算\(t=0\)时的旧稳态和\(t=t_c+1\)时的新稳态。
  3. 提供总量变量路径\(\{K_t^0,N_t^0\}_{t=1}^{t_c}\)的初始猜测值，通常可在新旧稳态间线性插值作为初始猜测。
  4. 计算转轨路径（计算细节见下节）。
  5. 新的路径\(\{K_t^1,N_t^1\}_{t=1}^{t_c}\)与上一次路径非常接近，则停止计算，否则以新路径按照线性更新或者其他方式更新之后，作为新的猜测值再次回到第四步，直至收敛。

2. 转轨路径计算：从猜测开始

因为有了转轨路径的猜测值，所以在转轨路径上要素价格、税收和养老金作为三个序列都是知道的。基于此，可以按照出生日期计算一代居民整个生命周期——这里为6期——的资本存量、劳动供给和消费，然后再在日历时间上对处于\(t\)期的各代人的\(k,n,c\)加总以获得总量数据。

具体而言，如果设定转轨时期为20期，我们可以利用如下居民的一阶条件来计算在第20期出生的人，其整个生命周期的资本存量、劳动供给和消费。 \[ \begin{align} &\gamma \frac{c_{t+s-1}^s+\psi}{1-n_{t+s-1}^s}=(1-\tau_{t+s-1})w_{t+s-1},s=1,2,3,4\\ &\frac{1}{\beta}=\frac{(c_{t+s}^{s+1}+\psi)^{-\eta}(1-n_{t+s}^{s+1})^{\gamma(1-\eta)}}{(c_{t+s-1}^s+\psi)^{-\eta}(1-n_{t+s-1}^s)^{\gamma(1-\eta)}}(1+r_{t+s}),s=1,2,\cdots,5 \end{align} \] 其中\(s\)是年龄。评论：

  • 在前面稳态计算时，我们使用的标记是\(c_t^s\)，但这里用了\(c_{t+s-1}^s\)。在计算上这没有什么影响，主要的用途在于要考虑\(s\)从1到4的变化，从而上标和下标都要变化。
  • 第一个条件只需要\(s=1,\cdots,4\)是因为该条件是针对工作居民的，退休居民无此一阶条件。第二个条件是工作居民和退休居民都应满足的，之所以\(s=5\)而没有\(s=6\)是因为一旦\(s=6\)就会有\(c^7\)，而居民活不过6期。同时注意到\(n^5=n^6=0\)。

那么在第20期出生的人，他一生应该满足的条件如下， \[ \begin{align} &\gamma \frac{c_{20}^1+\psi}{1-n_{20}^1}=(1-\tau_{20})w_{20}\\ \vdots\\ &\gamma \frac{c_{23}^4+\psi}{1-n_{23}^4}=(1-\tau_{23})w_{23}\\ &\frac{1}{\beta}=\frac{(c_{21}^{2}+\psi)^{-\eta}(1-n_{21}^{2})^{\gamma(1-\eta)}}{(c_{20}^1+\psi)^{-\eta}(1-n_{20}^1)^{\gamma(1-\eta)}}(1+r_{21})\\ \vdots\\ &\frac{1}{\beta}=\frac{(c_{25}^{6}+\psi)^{-\eta}(1-n_{25}^{6})^{\gamma(1-\eta)}}{(c_{24}^5+\psi)^{-\eta}(1-n_{24}^5)^{\gamma(1-\eta)}}(1+r_{25}) \end{align} \]

为了利用在第1期和第7期的资本存量为0的条件，可以把上式中的消费用资源约束条件替换掉。即， \[ \begin{align} &\gamma \frac{(1-\tau_{20}w_{20})n_{20}^1+(1+r_{20})k_{20}^1-k_{21}^2+\psi}{1-n_{20}^1}=(1-\tau_{20})w_{20}\tag{1}\\ \vdots\tag{..}\\ &\gamma \frac{(1-\tau_{23}w_{23})n_{23}^4+(1+r_{23})k_{23}^4-k_{24}^5+\psi}{1-n_{23}^4}=(1-\tau_{23})w_{23}\tag{4}\\ &\frac{1}{\beta}=\frac{((1-\tau_{21}w_{21})n_{21}^2+(1+r_{21})k_{21}^2-k_{22}^3+\psi)^{-\eta}(1-n_{21}^{2})^{\gamma(1-\eta)}}{((1-\tau_{20}w_{20})n_{20}^1+(1+r_{20})k_{20}^1-k_{21}^2+\psi)^{-\eta}(1-n_{20}^1)^{\gamma(1-\eta)}}(1+r_{21})\tag{5}\\ \vdots\tag{..}\\ &\frac{1}{\beta}=\frac{((1-\tau_{25}w_{25})n_{25}^6+(1+r_{25})k_{25}^6+\psi)^{-\eta}(1-n_{25}^{6})^{\gamma(1-\eta)}}{((1-\tau_{24}w_{24})n_{24}^5+(1+r_{24})k_{24}^5-k_{25}^6+\psi)^{-\eta}(1-n_{24}^5)^{\gamma(1-\eta)}}(1+r_{25})\tag{9} \end{align} \] 注意这个方程组第一个一阶条件体现为4个方程，第二个一阶条件体现为5个方程。而我们要求解的变量为\(\{k_{21}^2,\cdots,k_{25}^6,n_{20}^1,\cdots,n_{23}^4\}\)共9个，注意\(k_{20}^1=k_{26}^7=0\)。因此，最终问题归结为解一个非线性方程组。

这是在第20期出生的人，我们可以照此类推计算第19,18,\(\cdots\),1期出生的人。但对于在0期以及0期以前出生的人，需要有一点点修改。因为政策冲击是在第1期，那么在\(t=0\)他出生那一年并不知道下一期政策会改变，因此在\(t=1\)也即\(s=1\)时，他的劳动供给和资本存量都将是旧稳态值，\(s>1\)时劳动供给和资本存量才是新的路径上的值。这当然会带来工资、利率等参数的变化。这是对\(t=0\)期出生的人利用(1)-(9)式计算时需要注意的。对于\(t=-4,-3,\cdots,-1\)出生的，有类似的计算。但没有必要计算\(t=-5\)期出生的人，因为等到了\(t=1\)时，他已经不存在了。

当我们计算了每一代出生居民其整个生命周期的资本和就业后，就可以在日期\(t\)上对该时点上的各代居民进行汇总得到新的总量资本和就业， \[ K_t=\frac{1}{6}\sum_{s=1}^6k_t^s,\hspace{2em} N_t=\frac{1}{4}\sum_{s=1}^6n_t^s \]

3. 更新的方式

获得新的转轨路径值以后，有多种方式可以更新：

  1. 与计算稳态时的更新一样，进行线性更新。但当模型日益复杂以后，线性更新可能不会收敛，因此我们需要下面的新的更新算法。
  2. 通过解非线性方程组的方式进行更新。比如前述转轨路径算法中，我们将第\(i\)次迭代值设为\(K_1^i,K_2^i,\cdots,K_{20}^i,N_1^i,N_2^i,\cdots,N_{20}^i\)，通过前述计算会有一个新的值，我们记为\(K_1^{i*},K_2^{i*},\cdots,K_{20}^{i*},N_1^{i*},N_2^{i*},\cdots,N_{20}^{i*}\)，那么我们可以构造一个如下的非线性方程组，这个方程组是输入原有路径上的猜测值，输出新更新的值和原来值的差。

\[ g(K_1^i,K_2^i,\cdots,K_{20}^i,N_1^i,N_2^i,\cdots,N_{20}^i)= \begin{bmatrix} K_1^{i*}-K_1^i\\ K_2^{i*}-K_2^i\\ \cdots\\ K_{20}^{i*}-K_{20}^i\\ N_1^{i*}-N_1^i\\ N_2^{i*}-N_2^i\\ \cdots\\ N_{20}^{i*}-N_{20}^i \end{bmatrix}=0 \]

解这个方程组就能保证\(K^{i*}\)与\(K^i\)最终是无差异的，解这个方程组可以用牛顿法也可以用Broyden算法，后者一般速度会更快一些。

4. 代码实现

4.1 计算新旧稳态

当替代率\(\zeta=0.3\)时，计算模型的稳态，然后替代率下降到0.2，计算新的稳态。稳态的计算方法与60代确定性世代交叠模型稳态求解相同。我们将计算代码贴出如下。

library(pacman)
p_load(patchwork,ggplot2,matlab,rootSolve,nleqslv,magrittr,signal)
# 稳态计算：值函数迭代方法
# parameter
if (FALSE){
  phi <- 0.8    # updating parameter for method 1
  tw <- 4
  tr <- 2
  betax <- 0.9         # discount factor
  r <- 0.2         # initial value of the interest rate
  s <- 2            # coefficient of relative risk aversion
  alp <- 0.3        # production elasticity of capital
  zeta <- 0.3        # replacement ratio
  del <- 0.4          #/* rate of depreciation */
  gam <- 2            #/* disutility from working */
  psi <- 0.001      #/* parameter of utility function */
  # save.image(file = 'data-raw/para.rdata')
} else load('data-raw/para.rdata')
zeta <- 0.2      # 新的养老金替代率

tau <- zeta/(2 + zeta)    #/* income tax rate */
na <- 101          #/* number of grid points on assets */
a <- seq(0, 1, length.out = na)   #/* asset grid */

#initialization
nbar <- 0.2
kbar <- (alp/(r + del))^(1/(1-alp))*nbar

kold <- 100
nold <- 2

# agents' policy function in steady state
copt <- aopt <- zeros(tw + tr,1)  # optimal asset，and consumption
nopt <- 0.3*ones(tw,1)    # optimal labor supply

q <- 0
crit <- 1 + 1e-4
while(q < 30 & crit > 1e-4){
  q <- q + 1
  print(q)
  crit <- abs((kbar - kold)/kbar)

  w <- (1 - alp)*kbar^alp*nbar^(-alp)
  r <- alp*kbar^(alp-1)*nbar^(1 - alp) - del

  pen <- zeta*(1-tau)*w*nbar*3/2
  kold <- kbar
  nold <- nbar

  # 退休居民
  vr <- zeros(na,tr)  # 值函数
  aropt <- zeros(na,tr)  # 最优资本存量
  cropt <- zeros(na,tr)   # 最优消费
  # 第6期
  cropt[,tr] <- a*(1+r) + pen # 预算约束得到
  vr[,tr] <- utility(a*(1+r) + pen,1)
  for (i in (tr-1):1) {
    m0 <- 1
    for (LL in 1:na) {
      k0 <- a[LL]
      ans <- decise_range(a = a, value = value1, m0 = m0, vrnext = vr[,i+1],k0 = k0,pen = pen)
      # 格点搜索起点的更新
      m0 <- ans[4]
      aropt[LL,i] <- optvar(value1, scope = ans, a = a, vrnext = vr[,i+1], k0 = k0, pen = pen)
      vr[LL, i] <- value1(aropt[LL,i],a,vr[,i+1],k0=k0,pen=pen)
      cropt[LL,i] <- (1+r)*a[LL]+pen-aropt[LL,i]
    }
  }

  # 工作居民
  vw <- zeros(na,tw)
  awopt <- ones(na,tw)
  cwopt <- zeros(na,tw)
  nwopt <- zeros(na,tw)

  for (i in seq(tw,1)){
    if (i == tw){# 年轻人的最后一期就是老年人的第一期
      vw0 <- vr[,1]
    }else{
      vw0 <- vw[,i+1]
    }
    # 格点搜索的起点
    m0 <- 1

    for(LL in 1:na){
      k0 <- a[LL]
      ans <- decise_range(a = a, value = value2, m0 = m0, vrnext = vw0, k0 = k0,
                          tau = tau, w = w)
      m0 <- ans[4]
      k1 <- awopt[LL,i] <- optvar(value2, scope = ans, a = a, vrnext = vw0, k0 = k0,
                                  tau = tau, w = w, gam = gam)

      # 避免劳动力供给小于0
      n <- 1/(1 + gam)*(1 - gam/((1-tau)*w)*(psi+(1+r)*k0-k1))
      if (n < 0)  n <- 0

      cwopt[LL,i] <- (1-tau)*w*n+(1+r)*k0-k1
      nwopt[LL,i] <- n
      vw[LL,i] <- value2(awopt[LL,i],a = a,vrnext = vw0,k0 = k0, tau = tau, w = w)
    }
  }

  # 获得政策函数后，计算资本存量从0开始的演进路径
  # 变量初始化
  cgen <- kgen <- zeros(tw + tr,1)
  ngen <- zeros(tw,1)

  kgen[1] <- 0

  # 从0开始,资本存量演进，最后一期为0
  for (j in 1:(tw + tr-1)){
    if (j <= tw){
      kgen[j+1] <- interp1(a,awopt[,j],kgen[j],extrap = T)
      n <- 1/(1+gam)*(1-gam/((1-tau)*w)*(psi+(1+r)*kgen[j]-kgen[j+1]))
      ngen[j] <- n
      cgen[j] <- (1-tau)*w*n + (1+r)*kgen[j] - kgen[j+1]
    } else {
      kgen[j+1] <- interp1(a,aropt[,j-tw],kgen[j],extrap = T)
      cgen[j] <- (1+r)*kgen[j] + pen - kgen[j+1]
    }
  }
  cgen[tw+tr] <- (1+r)*kgen[tw + tr] + pen # 最后一期的消费

  knew <- mean(kgen)
  kbar <- phi*kold + (1-phi)*knew
  nnew <- mean(ngen)*2/3 # workers have share 2/3 in total population
  nbar <- phi*nold + (1-phi)*nnew
}

# 保存稳态下的个体最优资本存量
if (zeta == 0.3){
  wold <- w
  rold <- r
  penold <- pen
  tauold <- tau
  aoptold <- kgen
  noptold <- ngen
  coptold <- cgen
  kbarold <- kbar
  nbarold <- nbar
  save(wold, rold, penold, kbarold, nbarold, aoptold, noptold,tauold,coptold, file = 'data-raw/varold.rdata')
} else if (zeta == 0.2){
  wnew <- w
  rnew <- r
  pennew <- pen
  taunew <- tau
  aoptnew <- kgen
  noptnew <- ngen
  coptnew <- cgen
  kbarnew <- kbar
  nbarnew <- nbar
  save(wnew, rnew, pennew, taunew, kbarnew, nbarnew, aoptnew, noptnew, coptnew, file = 'data-raw/varnew.rdata')
}

这段代码做了几个特殊的处理，说明如下：

  • 把参数设置好保存在了para.rdata中，后续计算转移路径时调用。第一次运行的时候，要更改参数设置中条件语句的FALSE选项为TRUE，把参数保存好。
  • 参数设置后的第一行语句zeta = 0.2注释掉就是旧稳态计算，不注释新稳态计算。
  • 最后一大段代码是保存新旧稳态下的内生、外生变量，以便后续转移路径调用。

ktold <- seq(kbarold, kbarnew,length.out = nt)
ntold <- seq(nbarold, nbarnew, length.out = nt)

基于该总量猜测值，可以计算路径上的工资、利率、税收和养老金，

wt <- (1-alp)*ktold^alp*ntold^(-alp)

taut <- ones(nt,1)*taunew
pent <- 0.2*(1-taunew)*wt*ntold*3/2

基于路径上的工资、利率、税收和养老金，可以计算在第-4，-3，\(\cdots\)，20期出生的个体，每个个体他一生6期中面临的工资、利率、税收和养老金序列。或者可以设计一个函数，该函数通过输入他出生的日期，得到他一生6期的工资等序列，

wrpentau <- function(tt, wt,rt,pent,taut,wold,rold,penold,tauold,wnew,rnew,pennew, taunew, nt = 20){
  wseq <- rseq <- penseq <- tauseq <- numeric(6)
  if (tt > nt-5){ # 在16,17,...,20期
    # agent is alive after period nt
    wseq[1:(nt-tt+1)] <- wt[tt:nt]
    wseq[(nt-tt+2):6] <- ones(6-nt+tt-1,1)*wnew
    rseq[1:(nt-tt+1)] <- rt[tt:nt]
    rseq[(nt-tt+2):6] <- ones(6-nt+tt-1,1)*rnew
    penseq[1:(nt-tt+1)] <- pent[tt:nt]
    penseq[(nt-tt+2):6] <- ones(6-nt+tt-1,1)*pennew
    tauseq[1:(nt-tt+1)] <- taut[tt:nt]
    tauseq[(nt-tt+2):6] <- ones(6-nt+tt-1,1)*taunew
  } else if (tt < 1){
    wseq[1:(1-tt)] <- ones(-tt+1,1)*wold
    wseq[(2-tt):6] <- wt[1:(4+tt+1)]
    rseq[1:(1-tt)] <- ones(-tt+1,1)*rold
    rseq[(2-tt):6] <- rt[1:(4+tt+1)]
    penseq[1:(1-tt)] <- ones(-tt+1,1)*penold
    penseq[(2-tt):6] <- pent[1:(4+tt+1)]
    tauseq[1:(1-tt)] <- ones(-tt+1,1)*tauold
    tauseq[(2-tt):6] <- taut[1:(4+tt+1)]
  } else { # 在15,14,...,1期
    wseq[1:6] <- wt[tt:(tt+5)]
    rseq[1:6] <- rt[tt:(tt+5)]
    penseq[1:6] <- pent[tt:(tt+5)]
    tauseq[1:6] <- taut[tt:(tt+5)]
  }
  return(data.frame(wseq = wseq, rseq = rseq, penseq = penseq, tauseq = tauseq))
}

当个体拥有了他的工资、利率、税收和养老金序列后，就可以通过解(1)至（9）式的非线性方程组计算其一生资本存量和劳动供给序列。那么，首先我们要为（1）至（9）式非线性方程组写个函数，

rftr <- function(x, tauseq, wseq, rseq, penseq,tt,
                 coptold, aoptold, noptold, betax = 0.9, psi = 0.001, gam = 2){
  # x 前面5个是下一期的资本存量, 后面4个是劳动力供给
  # y 理论上是9个非线性方程组的左边，为0
  y <- x
  k2 <- x[1]
  k3 <- x[2]
  k4 <- x[3]
  k5 <- x[4]
  k6 <- x[5]
  n1 <- x[6]
  n2 <- x[7]
  n3 <- x[8]
  n4 <- x[9]
  # 为第一个一阶条件准备消费的数据
  c1 <- (1-tauseq[1])*wseq[1]*n1-k2
  c2 <- (1-tauseq[2])*wseq[2]*n2+(1+rseq[2])*k2-k3
  c3 <- (1-tauseq[3])*wseq[3]*n3+(1+rseq[3])*k3-k4
  c4 <- (1-tauseq[4])*wseq[4]*n4+(1+rseq[4])*k4-k5
  c5 <- penseq[5]+(1+rseq[5])*k5-k6
  c6 <- penseq[6]+(1+rseq[6])*k6
  # 第一个一阶条件
  y[1] <- gam*(c1+psi)-(1-tauseq[1])*wseq[1]*(1-n1)
  y[2] <- gam*(c2+psi)-(1-tauseq[2])*wseq[2]*(1-n2)
  y[3] <- gam*(c3+psi)-(1-tauseq[3])*wseq[3]*(1-n3)
  y[4] <- gam*(c4+psi)-(1-tauseq[4])*wseq[4]*(1-n4)
  # browser()

  # in period 1: unexpected change, therefore, prior to period 1, all agents
  # behave as in the old steady state
  # y[5] to y[9]是第二个一阶条件
  if (tt<=0){
    c1 <- coptold[1]
    y[1] <- n1-noptold[1] # 即n1 = noptold[1]，比如第0期出生的，该期他仍然处于旧稳态中
    y[5] <- k2-aoptold[2] # 这是第二个一阶条件的第一个方程
  } else {
    y[5] <- 1/betax-uc(c2,1-n2)/uc(c1,1-n1)*(1+rseq[2])
  }

  if (tt<=-1){
    c2 <- coptold[2]
    y[2] <- n2-noptold[2]
    y[6] <- k3-aoptold[3]
  } else {
    y[6] <- 1/betax-uc(c3,1-n3)/uc(c2,1-n2)*(1+rseq[3])
  }

  if (tt<=-2){
    c3 <- coptold[3]
    y[3] <- n3-noptold[3]
    y[7] <- k4-aoptold[4]
  } else {
    y[7] <- 1/betax-uc(c4,1-n4)/uc(c3,1-n3)*(1+rseq[4])
  }

  if (tt<=-3){
    c4 <- coptold[4]
    y[4] <- n4-noptold[4]
    y[8] <- k5-aoptold[5]
  } else {
    y[8] <- 1/betax-uc(c5,1)/uc(c4,1-n4)*(1+rseq[5])
  }

  if (tt<=-4){
    c5 <- coptold[5]
    y[9] <- k6-aoptold[6]
  } else {
    y[9] <- 1/betax-uc(c6,1)/uc(c5,1)*(1+rseq[6])
  }
  return(y)
}

注意该函数调用的uc是效用函数的导数，

uc <- function(x,y, psi = 0.001){
  # 效用函数对c的导数
  # x: c
  # y: 闲暇leisure
  (x+psi)^(-s)*y^(gam*(1-s))
}

aggkn <- function(tp, ktnew, ntnew, aopt, nopt){
  i <- 0
  while (!(tp == nt | i == 6)){# 只要到了20期，无论你是几岁都停止计算。或者到了7岁也停止计算
    tp <- tp+1
    i <- i+1
    if (tp > 0){
      ktnew[tp] <- ktnew[tp]+1/6*aopt[i] # 每期有1-6岁，所以每个岁数占1/6
      if (i<=4){
        ntnew[tp] <- ntnew[tp]+1/6*nopt[i]
      }
    }
  }
  return(list(ktnew = ktnew, ntnew = ntnew))
}

tp是加总变量的日期，在该期计算各代出生的个体变量在该期的加总值。ktnew,ntnew是上一次汇总后的值，aopt,nopt是tt期出生的个体变量整个生命周期的值。

上述过程就完成了一次从转轨路径猜测值到转轨路径新的计算值的一次更新。如果我们新计算的转轨路径和原来的转轨路径差距还比较大，那我们还要把上述过程再来一遍，直到新的路径与旧的路径几乎无差异。

那么，为了让代码更加富有逻辑性，我们可以把从旧轨迹猜测值到计算出新轨迹值这个过程封装成一个函数，然后就可以基于这个函数反复运算直到旧轨迹和新轨迹几乎无差异。这个新的函数可以写成，

old2new <- function(x, nt,
                    wold, rold, penold, tauold,
                    wnew, rnew, pennew, taunew,
                    nle = FALSE, alp = 0.36, del = 0.1, zeta1 = 0.2){

  # 分配旧路径的值
  ktold <- x[1:nt]
  ntold <- x[(nt+1):(2*nt)]

  wt <- (1-alp)*ktold^alp*ntold^(-alp)
  rt <- alp*ktold^(alp-1)*ntold^(1-alp)-del
  taut <- ones(nt,1)*taunew
  pent <- zeta1*(1-taunew)*wt*ntold*3/2

  # 按照出生日期循环前的初始化
  ktnew <- zeros(nt,1)
  ntnew <- zeros(nt,1)
  nopt_all <- aopt_all <- vector('list')
  tt <- nt + 1
  while (tt != -4) {
    tt <- tt-1
    # 基于出生期tt获得个体生命期的w,r,pen,tau序列
    indseq <- wrpentau(tt = tt, wt = wt, rt = rt, pent = pent, taut = taut,
                       wold = wold, rold = rold, penold = penold, tauold = tauold,
                       wnew = wnew, rnew = rnew, pennew = pennew, taunew = taunew, nt = nt)

    # 解个体生命期的非线性方程组
    x0 <- c(aopt[2:6],nopt[1:4]) # 解非线性方程组的初值
    xf <- multiroot(start = x0, f = rftr,tauseq = indseq$tauseq, wseq = indseq$wseq, rseq = indseq$rseq,
                  penseq = indseq$penseq, tt = tt, coptold = coptold, aoptold = aoptold,
                  noptold = noptold)$root
    aopt[1] <- 0 # 任何个体第一期资本存量为0
    aopt[2:6] <- xf[1:5]
    nopt[1:4] <- xf[6:9]

    # 这一部分对每期的1-6岁的居民资产和劳动进行汇总
    ans <- aggkn(tp = tt-1, ktnew, ntnew, aopt, nopt)
    ktnew <- ans$ktnew
    ntnew <- ans$ntnew

    # 存下各年龄段居民的资本和劳动
    aopt_all[[as.character(tt)]] <- aopt
    nopt_all[[as.character(tt)]] <- nopt
  }

  # 该函数用来解非线性方程组时返回与x长度相同的向量，其他情况返回更多数据
  ifelse (nle,
          return(c(ktnew, ntnew)-x),
          return(list(ktnew = ktnew, ntnew = ntnew, aopt_all = aopt_all, nopt_all = nopt_all)))
}

该函数说明如下：

  • x是输入的旧轨迹，它把\(K\)和\(N\)的轨迹连成一个向量是为了方便后面的通过解非线性方程组的更新策略。
  • nt是转轨路径的长度。wold, rold, penold, tauold,wnew, rnew, pennew, taunew是新旧稳态下得到的工资、利率、养老金和税收数值。其他输入是模型的参数。
  • 该函数输出有两种输出方式，取决于nle的设置，如果是TRUE，就是通过解非线性方程组来更新的话，它只会输出一个新的轨迹向量。否则，它会输出更多变量值。
