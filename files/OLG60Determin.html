模型设置

居民部门

先对居民进行建模，假设经济体一共\(T+T^R=60\)代人，前\(T=40\)代工作，后\(T^R=20\)代退休，60代人的总数量标准化为1，也就是每代人数量为\(\frac{1}{60}\)。

\(s\)岁的工作的人提供劳动力数量为\(n_t^s\)，享受的闲暇为\(l_t^s=1-n_t^s\)，在\(T=40\)年后退休，劳动供给\(n_t^s=0,s>T\)。居民的最优规划写为， \[ \sum_{s=1}^{T+T^R}\beta^{s-1}u(c_{t+s-1}^s,l_{t+s-1}^s)=\frac{[(c_{t+s-1}+\psi)l_{t+s-1}^\gamma]^{1-\eta}-1}{1-\eta} \] 但工作和退休居民的约束有所不同，分别为， \[ \begin{align} k_{t+s}^s=&(1+r_t)k_t^s+(1-\tau_t)w_tn_t^s-c_t^s,\;\;&s=1,\cdots,T\\ k_{t+s}^s=&(1+r_t)k_t^s+b-c_t^s,\;\;&s=T+1,\cdots,T^R\\ \end{align} \] 其中\(\tau_t\)是所得税税率，\(b\)是养老金。

厂商

对于厂商采取\(Y_t=N_t^{1-\alpha}K_t^\alpha\)的形式生产，利润最大化意味着， \[ w_t=(1-\alpha)K_t^\alpha N_t^{-\alpha}\\ r_t=\alpha K_t^{\alpha-1}N_t^{1-\alpha}-\delta \] 其中\(\delta\)是折旧率。

政府

政府只做一件事，就是保持养老金的预算平衡， \[ \tau_tw_tN_t=\frac{T^R}{T+T^R}b\tag{1}\label{eqb} \] 其中\(N_t=\sum_{s=1}^T\frac{n_t^s}{T+T^R}\)是劳动供给总量。因为\(b\)是每一代老年人的养老金数额，每代人的数量为\(\frac{1}{T+T^R}\)，因此全部老年人的养老金数额就是\(\frac{T^R}{T+T^R}b\)。但很多时候，文献喜欢用替代率而不是\(b\)来表述养老金取值，所谓替代率是指养老金与工资收入的比率，即\(\zeta=\frac{b}{(1-\tau)w\bar{n}},\bar{n}\)是平均劳动供给，因为数值运算时，退休的劳动供给为0，所以\(\bar{n}\)往往计算的是工作人员的平均劳动供给，对于本例而言\(\bar{n}=\frac{\sum_{s=1}^{40} n_t^s}{40}\)，这就意味着\(40\bar{n}=60N_t\)。当给定替代率\(\zeta\)时， \[b=\zeta(1-\tau)w\bar{n}=\zeta(1-\tau)w\cdot N_t\cdot 3/2\tag{2}\label{eqb2}\]

在均衡时，整个模型需要在个体和总体之间保持一致性。注意，因为每代人数量为1/60，因此加总劳动力时，在算法上就体现为求均值了。 \[ N_t=\sum_{s=1}^{T}\frac{n_t^s}{T+T^R}\\ K_t=\sum_{s=1}^{T+T^R}\frac{k_t^s}{T+T^R} \]

模型求解

这类OLG模型求解，最好用值函数迭代，因为个体的最后一期的资本存量是0，所以在寻找个体最优资本存量时，这种值函数迭代的方法不需要像往常一样先猜一个初始的值函数初值然后再通过循环直至值函数收敛。它只需要从最后一期开始往前迭代，就能得到每一期最优的资本存量。令\(V^s(k_t^s,K_t,N_t)\)是\(s\)岁居民在财富\(k_t^s\)的值函数， \(K_t,N_t\)分别是资本存量和就业总量，值函数可以写成， \[V^s(k_t^s,K_t,N_t)= \begin{cases} \max_{k_{t+1}^{s+1},c_t^s,l_t^s}[u(c_t^s,l_t^s)+\beta V^{s+1}(k_{t+1}^{s+1},K_{t+1},N_{t+1})],s=1,\cdots,T\\ \max_{k_{t+1}^{s+1},c_t^s,l_t^s}[u(c_t^s,1)+\beta V^{s+1}(k_{t+1}^{s+1},K_{t+1},N_{t+1})],s=T+1,\cdots,T^R\\ \end{cases} \] 注意值函数的状态变量之所以包含总体变量\(K_t,N_t\)，是因为我们要用它来计算工资\(w_t\)和利率\(r_t\)。

具体的算法，我们勾勒如下，

  1. 给定稳态时资本存量和就业总量\(K\)和\(N\)的初始猜测值。
  2. 基于该猜测值计算\(w,r,\tau\)。
  3. 给定资本存量终值\(k^{61}=0\)，沿着\(s=60,59,\cdots,1\)往前迭代计算每代人的最优消费、储蓄和就业。
  4. 计算\(K,N\)。
  5. 更新\(K,N\)并返回到第2步直到迭代收敛。

我们仍然像在RBC值函数迭代算法一样，先生成资产格点，然后针对每一个资产格点，计算下一期最优的资本存量和最优消费。对于退休人员而言，从其预算约束可知，因为最后一期的资本存量\(k_{61}^{60}=0\)，所以很容易得到其在第60期的最优消费，一旦有了最优消费，利用消费函数就可以计算第60期的最优值函数。

cropt[,tr] <- a*(1+r)+pen # 预算约束得到
utility <- function(x,y){
  if (s==1){ # eta 为1时的效用函数
    return(log(x+psi) + gam*log(y))
  } else{
    return((((x+psi) * (y**gam))^(1-s) - 1)/(1-s))
  }
}
vr[,tr] <- utility(a*(1+r)+pen,1)

通过贝尔曼方程，当已知下一期的值函数时，计算最优资本存量已经轻车熟路。类似前面RBC模型的值函数迭代，把寻找下一期最优资本存量范围的函数decise_range载入，

decise_range <- function(a, value, m0, vrnext, neg = -1e10, ...){
  # triple ax, bx, cx: [ax,bx] bracket the maximum of Bellman eq.
  # 初始设置故意使得cx < bx < ax
  ax <- 0
  bx <- -1
  cx <- -2
  v0 <- neg
  m <- max(0,m0-2)
  # browser()

  # locate ax <= a' <= cx that bracket the maximum
  while (ax>bx | bx>cx){
    m <- m+1
    v1 <- value(anext = a[m], a = a, vrnext = vrnext,...)

    # 如果V1一直比V0大，意味着函数的拐点没有达到，所以m还要往前递进
    # 直到V1比V0小，意味着拐点就在m-1和m之间了，此时就可以给cx赋值
    if (v1 > v0){# new value at lower bound a[0]
      if (m==1){
        ax <- a[m]
        bx <- a[m]
      } else {
        bx <- a[m]
        ax <- a[m-1]
      }
      v0 <- v1
      m0 <- m   # monotonocity of the value function
    } else cx <- a[m]

    if (m == na){
      ax <- a[m-1]
      bx <- a[m]
      cx <- a[m]
    }
  }
  return(c(ax = ax, bx = bx, cx = cx, m0 = m0))
}

该函数的参数含义如下：

  • a是向量，表示资产格点。
  • value是值函数。
  • m0是格点的搜索的起始位置。随着从小到大的格点逐个搜索，这个位置会逐渐后移，因为值函数是单调的。
  • vrnext是下一期值函数具体取值的向量。
  • ...是传入value的其他参数。

该函数返回四个值，第一和第三个是下一期最优资本存量的上下界范围，第四个是下一次继续格点搜索时，搜索的起点。

接下来载入退休人员的值函数，

value1 <- function(anext, a, vrnext, k0, pen){

    return(-1e10)
  } else{
    return(utility(cc,1) + bt*interp1(a,vrnext,anext, extrap = T))

}

现在考虑工作居民的下一期最优资本存量、消费、劳动供给\(n\)和值函数取值。这里多了一个劳动供给的计算，注意到居民的一阶条件可以写成， \[ \begin{align} &\frac{u_l(c_t^s,l_t^s)}{u_c(c_t^s,l_t^s)}=\gamma \frac{c_t^s+\psi}{l_t^s}=(1-\tau_t)w_t\tag{1}\\ &\frac{1}{\beta}=\frac{u_c(c_{t+1}^s,l_{t+1}^s)}{u_c(c_t^s,l_t^s)}(1+r_{t+1})=\frac{(c_{t+1}^{s+1}+\psi)^{-\eta}(l_{t+1}^{s+})^{\gamma(1-\eta)}}{(c_t^s+\psi)^{-\eta}(l_t^s)^{\gamma(1-\eta)}}(1+r_{t+1})\tag{2} \end{align} \] 要计算稳态下的\(n\)，仅需第一个条件即可。对于工作的居民而言，计算稳态时去掉时间脚标，将资源约束代入以消掉\(c_t^s\)，有， \[ \begin{align} (1-\tau)w=&\gamma\frac{(1+r)k^s+(1-\tau)wn^s-k^{s+1}+\psi}{1-n^s}\\ \Longrightarrow (1-\tau)w(1-n^s)-\gamma (1-\tau)wn^s=&\gamma\left[(1+r)k^s-k^{s+1}+\psi\right]\\ \Longrightarrow -(1-\tau)wn^s-\gamma (1-\tau)wn^s =& \gamma[(1+r)k^s-k^{s+1}+\psi] -(1-\tau)w\\ \Longrightarrow n^s =&\frac{(1-\tau)w-\gamma[(1+r)k^s-k^{s+1}+\psi]}{(1-\tau)w(1+\gamma)}\\ =& \frac{1}{1+\gamma}\left(1-\frac{\gamma[(1+r)k^s-k^{s+1}+\psi]}{(1-\tau)w}\right) \end{align} \]

value2 <- function(anext, a, vrnext, k0, tau, w, bt = 0.96, psi = 0.001){
  # optimal labor supply implied by first-order condition
  n <- 1/(1+gam) * (1- gam/((1-tau)*w) * (psi+(1+r)*k0-anext))

  if (n<0){
    # corner solution 0<=n<=1
    n <- 0
  } else if (n >= 1){
    # corner solution
    n <- 1
  }

  cc <- (1-tau)*w*n + (1+r)*k0 - anext
  if (cc <= 0){
    return(-1e+10)
  }else{
    return(utility(cc,1-n) + bt*interp1(a,vrnext,anext, extrap = T))
  }
}

      }
      k1 <- awopt[LL,i]
      
      # 避免劳动力供给小于0
      n <- 1/(1+gam)*(1-gam/((1-tau)*w)*(psi+(1+r)*k0-k1))
      if (n < 0)  n <- 0

      cwopt[LL,i] <- (1-tau)*w*n+(1+r)*k0-k1
      nwopt[LL,i] <- n
      vw[LL,i] <- value2(awopt[LL,i],a = a,vrnext = vw0,k0 = k0, tau = tau, w = w)
    }
  }

现在我们已经得到了工作居民和退休居民在稳态下，今年资本存量和下一年最优资本存量之间的映射关系。那么此时就可以计算当资本存量的初值为0时，资本存量随着年龄的演进路径。而一旦获得资本存量的路径，则消费和劳动供给的路径也就确定了。

# 变量初始化
cgen <- kgen <- zeros(tw+tr,1)
ngen <- zeros(tw,1)

kgen[1] <- 0

# 从0开始,资本存量演进，最后一期为0
for (j in 1:(tw+tr-1)){
    if (j <= tw){
      kgen[j+1] <- interp1(a,awopt[,j],kgen[j],extrap = T)
      n <- 1/(1+gam)*(1-gam/((1-tau)*w)*(psi+(1+r)*kgen[j]-kgen[j+1]))
      ngen[j] <- n
      cgen[j] <- (1-tau)*w*n + (1+r)*kgen[j] - kgen[j+1]
    } else {
      kgen[j+1] <- interp1(a,aropt[,j-tw],kgen[j],extrap = T)
      cgen[j] <- (1+r)*kgen[j] + pen - kgen[j+1]
    }
}
cgen[tw+tr] <- (1+r)*kgen[tw+tr] + pen # 最后一期的消费

然后，加总资本存量、就业,同时更新旧的资本存量和就业。注意到此处的加总在算法上等价于求均值，同时，我们使用线性更新，就是原来的值占80%，新的值占20%，缓慢更新。

phi <- 0.8
knew <- mean(kgen)
kbar <- phi*kold + (1-phi)*knew
nnew <- mean(ngen)*2/3 # workers have share 2/3 in total population
nbar <- phi*nold + (1-phi)*nnew

至此，则完成了一次运算。只要新计算的资本存量和初始猜测的资本存量还存在距离，那么这样的运算就要再来一次。要实现这样的算法，我们当然要利用while循环达成。

因此，下面的代码就是对上述算法的while循环封装，直至收敛以获得最终不同年龄段的稳态值。

# 删除其他变量，仅保留6个函数
ls()[!(ls() %>% str_detect('dec|int|util|value1|value2|wage'))] %>% rm(list = .)
# 参数设置
bt <- 0.96         # 贴现因子 beta
r <- 0.045        # 利率初值
s <- 2            # 相对风险回避系数 eta
alp <- 0.36       # 资本产出份额
zeta <- 0.30       # 养老金替代率
delta <- 0.10     # 折旧率
tr <- 20          # 退休期
tw <- 40           # 工作期
gam <- 2          # 效用函数参数 gamma
psi <- 0.001         # 效用函数参数
tau <- zeta / (2+zeta)    # 均衡预算下的税率
phi <- 0.8 # 更新比例

na <- 200 # 资本存量格点数目
a <- seq(0,10,length.out = na)   # 生成0-10的资产格点
aeps <- (a[2]-a[1])/na     #  用来测试角点解

nbar <- 0.2 # 劳动总量猜测值
kbar <- (alp/(r+delta))^(1/(1-alp))*nbar # 基于劳动总量在RBC模型下的资本存量计算公式，可以作为一个初始猜测
nold <- 2
kold <- 100
nq <- 30 # 在算稳态时可能迭代的次数，可以先设大一些，后续更改
kq <-  zeros(nq,2)   # 生成一个变量，第一列用来存储迭代过程中的加总资本存量，第二列存劳动力

q <- 1
crit <- 1 + 1e-4 # 收敛精度初始设定
while (q < 30 & crit > 1e-4){ # 停止循环的条件是计算超过30次或者收敛精度达到
  q <- q + 1
  if(q %% 5 == 0) print(q)
  crit <- abs((kbar - kold)/kbar)
  
  w <- wage_rate(kbar, nbar)       # wage rate
  r <- interest_rate(kbar, nbar)   # interest rate
  pen <- zeta*(1-tau)*w*nbar*3/2    # pensions: balanced social security budget
  
  kold <- kbar
  nold <- nbar
  kq[q,1] <- kbar
  kq[q,2] <- nbar
  
  vr <- zeros(na,tr)  # 值函数
  aropt <- zeros(na,tr)  # 最优资本存量
  cropt <- zeros(na,tr)   # 最优消费
  # 第60期
  cropt[,tr] <- a*(1+r)+pen # 预算约束得到
  vr[,tr] <- utility(a*(1+r)+pen,1)
  
  # 退休居民
  for (i in  seq(tr-1,1)){# all ages i=T+TR,T+TR-1,..,T
    # 格点搜索的起点
    m0 <- 1

    for (LL in 1:na) {
      k0 <- a[LL]
      ans <- decise_range(a = a, value = value1, m0 = m0, vrnext = vr[,i+1],k0=k0,pen=pen)
      # 格点搜索起点的更新
      m0 <- ans[4]
      
      if (ans[1] == ans[2]){
        if (value1(ans[1], a = a, vrnext = vr[,i+1], k0=k0,pen=pen) > 
            value1(aeps, a = a, vrnext = vr[,i+1],k0=k0,pen=pen)){
          aropt[LL,i] <- 0
        }else {
          aropt[LL,i] <- optimise(value1, c(ans[1],ans[3]), a = a, vrnext =vr[,i+1],
                                  maximum = T,k0=k0,pen=pen)$maximum
        }
      } else if (ans[2] == ans[3]){
        if (value1(a[na],a = a, vrnext = vr[,i+1],k0=k0,pen=pen) > 
            value1(a[na]-aeps,a = a, vrnext = vr[,i+1],k0=k0,pen=pen)){
          aropt[LL,i] <- a[na]
        }else {
          aropt[LL,i] <- optimise(value1, c(a[na-1],kmax),a = a, vrnext =vr[,i+1],
                                  maximum = T,k0=k0,pen=pen)$maximum
        }
      } else {
        aropt[LL,i] <- optimise(value1,c(ans[1],ans[3]),a = a, vrnext =vr[,i+1],
                                maximum = T,k0=k0,pen=pen)$maximum
      }
      vr[LL, i] <- value1(aropt[LL,i],a,vr[,i+1],k0=k0,pen=pen)
      cropt[LL,i] <- (1+r)*a[LL]+pen-aropt[LL,i]
    }
  }
  
  # 工作居民
  vw <- zeros(na,tw)
  awopt <- ones(na,tw)
  cwopt <- zeros(na,tw)
  nwopt <- zeros(na,tw)
  
  for (i in seq(tw,1)){
    if (i == tw){# 年轻人的最后一期就是老年人的第一期
      vw0 <- vr[,1]
    }else{
      vw0 <- vw[,i+1]
    }

    # 格点搜索的起点
    m0 <- 1

    for(LL in 1:na){
      k0 <- a[LL]
      ans <- decise_range(a = a, value = value2, m0 = m0, vrnext = vw0, k0 = k0, tau = tau, w = w)
      m0 <- ans[4]

      if (ans[1]==ans[2]){# corner solution: a'=0?
        if (value2(ans[1], a = a,vrnext = vw0, k0 = k0, tau = tau, w = w) > 
            value2(aeps, a = a,vrnext = vw0,k0 = k0, tau = tau, w = w)){
          awopt[LL,i] <- 0
        } else{
          awopt[LL,i] <- optimise(value2,c(ans[1],ans[3]),a = a, vrnext = vw0, 
                                  maximum = T,k0 = k0, tau = tau, w = w)$maximum
        }
      } else if (ans[2] == ans[3]){# corner solution: a'=a[na-1]=kmax?
        if (value2(a[na],a = a, vrnext = vw0,k0 = k0, tau = tau, w = w) > 
            value2(a[na]-aeps, a = a,vrnext = vw0,k0 = k0, tau = tau, w = w)){
          awopt[LL,i] <- a[na]
        } else {
          awopt[LL,i] <- optimise(value2,c(a[na-1],a[length(a)]),a = a, vrnext = vw0, 
                                  maximum = T, k0 = k0, tau = tau, w = w)$maximum
        }
      } else {
        awopt[LL,i] <- optimise(value2,c(ans[1],ans[3]),a = a, vrnext = vw0, 
                                maximum = T, k0 = k0, tau = tau, w = w)$maximum
      }
      k1 <- awopt[LL,i]
      
      # 避免劳动力供给小于0
      n <- 1/(1+gam)*(1-gam/((1-tau)*w)*(psi+(1+r)*k0-k1))
      if (n < 0)  n <- 0

      cwopt[LL,i] <- (1-tau)*w*n+(1+r)*k0-k1
      nwopt[LL,i] <- n
      vw[LL,i] <- value2(awopt[LL,i],a = a,vrnext = vw0,k0 = k0, tau = tau, w = w)
    }
  }
  
  # 变量初始化
  cgen <- kgen <- zeros(tw+tr,1)
  ngen <- zeros(tw,1)
  
  kgen[1] <- 0
  
  # 从0开始,资本存量演进，最后一期为0
  for (j in 1:(tw+tr-1)){
    if (j <= tw){
      kgen[j+1] <- interp1(a,awopt[,j],kgen[j],extrap = T)
      n <- 1/(1+gam)*(1-gam/((1-tau)*w)*(psi+(1+r)*kgen[j]-kgen[j+1]))
      ngen[j] <- n
      cgen[j] <- (1-tau)*w*n + (1+r)*kgen[j] - kgen[j+1]
    } else {
      kgen[j+1] <- interp1(a,aropt[,j-tw],kgen[j],extrap = T)
      cgen[j] <- (1+r)*kgen[j] + pen - kgen[j+1]
    }
  }
  cgen[tw+tr] <- (1+r)*kgen[tw+tr] + pen # 最后一期的消费

  
  knew <- mean(kgen)
  kbar <- phi*kold + (1-phi)*knew
  nnew <- mean(ngen)*2/3 # workers have share 2/3 in total population
  nbar <- phi*nold + (1-phi)*nnew
}
